# /impl — 実装着手チェックリスト

このリポジトリで変更を加える前に必ず以下を実行する。

## 1. ドキュメント確認（必須）

```
DECISIONS.md の「禁止事項」セクションを読む
SPEC.md の関連 MUST 節を読む
DECISIONS.md の関連決定ログを読む
```

禁止事項に抵触しないかを確認してから次に進む。

## 2. 方針をユーザーに説明して合意を取る（必須）

実装の前に、変更の方針（何をどう変えるか・理由）を1〜3文でユーザーに伝え、
Go サインをもらってから着手する。

## 3. 実装順序（固定）

1. **ドキュメント更新** — DECISIONS.md / SPEC.md / CLAUDE.md / ROADMAP.md を実態に合わせる
2. **失敗テスト作成** — SPEC §11 のテスト ID を根拠に書く。`npm test` でレッドを確認
3. **実装** — テストがグリーンになるまで進める
4. **型チェック** — `npm run build` を通す

## 4. 完了確認

```bash
npm test       # 全テストパス
npm run build  # 型エラーなし
```

## 禁止（DECISIONS.md より抜粋）

- キーワードマッチでの直行ルーティング
- ヒューリスティックによるエージェントのスキップ
- `memo_write` 成功時に `episodes` へ直接追記
- フィルタを理由に元データ（LanceDB・state.json）を削除・変更
- ロールごとに想起・会話ログの入れ方を変える
- プロンプトに行動指示を書いて構造で解決すべき問題を回避する
