# 自律ホームエージェント コンセプト設計書 v0.4

---

## コアコンセプト

「意思決定・実行・言語化・内省」を構造的に分離し、8Bモデル一本で破綻なく自律駆動する最小認知アーキテクチャ。分離脳の左脳モデルに倣い、言語化と内省は行動の原因ではなく結果として生成される。意識はリーズニングプロセスにアクセスできず、行動と発話の結果だけを受け取って後付けで解釈する、という人間の認知モデルを設計原理とする。

### 認知の3層

ターン全体は **入力 → 判断 → 行動/出力** の3層で構成する。

- **入力層（プリプロセス）**: センサー・永続記憶・作業記憶から揮発コンテキスト（`TurnContext`）を組み上げる。アーキテクチャの起点
- **判断層（ジャッジ）**: 抽象カテゴリ + intent + REPLY + NEXT_STATE を決める
- **行動・出力層**: サブエージェントが行動を実行し、言語野（Reply）と内省（Memory）を出力する

「アクション意思決定の2段ディスパッチ（ジャッジ→サブエージェント）」は、この行動・出力層の**内側**の話であり、3層アーキテクチャとは別レベルの概念。詳細は [ACTION-DESIGN.md](./ACTION-DESIGN.md) のレイヤモデルを参照。

## アーキテクチャ全体図

```
[外部センサー群]
  ペケッタータイムライン / Webカメラ / システム時刻 / その他IoT
       ↓ 常時書き込み
[永続層]
  エピソード記憶（LanceDB・ベクトル検索）
  意味記憶（エピソードからの蒸留・後回し）
  各センサー生ログ
       ↓ ターン開始時に想起・引用
[プリプロセス]
  現在Stateに応じて各チャンネルをフィルタ・要約し
  メイン脳が読めるサイズのコンテキストを組み上げる
       ↓
[コンテキスト]（揮発・毎ターン生成・ターン終了後破棄）
  作業記憶（直近会話履歴）
  + 想起されたエピソード記憶
  + 必要な外部センサー状態
       ↓
[ジャッジくん / 8B]
  3スロットを埋めて即座に引っ込む冷徹な裁判官
       ↓
[ポストプロセス]
  前段：行動くん
    ACTION の kind に応じてサブモジュールへディスパッチ（自身は LLM を呼ばない）
    失敗時はエラーログを残して言語野くんに渡す
  中段：言語野くん（8B）
    今ターンの全情報を受け取りキャラクタールールに従ってセリフ生成・出力
    ハートビートで REPLY=false でも ACTION 成功時は独り言を生成する
  後段：内省くん（8B）
    発話結果だけを受け取り一人称の内省文を生成
    ジャッジくんの決定プロセスもツールログも見えない
    「自分が何を言ったか」から「なぜそう思ったか」を推測して書く
       ↓
[エピソード記憶への書き込み]
  内省くんの出力をLanceDBに追記（永続層へ）
       ↓
[State遷移]
  NEXT_STATEの指示通りにStateを切り替えてターン完了
  コンテキストを破棄
```

## 主要コンポーネント詳細

### State
- `対話`：ユーザーが発話した、または自分が喋っている状態
- `静穏`：ハートビートによる潜伏状態

2値を指示するがバリデーションはしない。逸脱値はログ出力して観察する。ハートビート・思索モード実装時にログを参照してState設計を拡張する。

### コンテキスト（揮発・使い捨て）
毎ターン開始時にプリプロセスが生成し、ターン終了後に破棄する一時的な作業台。人間のワーキングメモリに相当し、永続化しない。

流れ込むチャンネル：
- 作業記憶（直近の会話履歴）
- 想起されたエピソード記憶（LanceDBからベクトル検索で引用。要約・ぼかし OK）
- 外部センサー状態（初期はシステム時刻のみ）

**記憶とメモの違い**: エピソード記憶（LanceDB）はふんわり思い出す（LLM 要約・グラデーション可）。共有メモ（`data/notes/`）は意図して残した全文をそのまま保持し、読み出し時に LLM で改変しない。

### プリプロセス
知的判断をしない濾過マシナリー。現在のStateに応じて各チャンネルから必要な情報だけを引き出し、フィルタ・要約してコンテキストを組み上げる。

### ジャッジくん（8B）
ロールプレイも言語化も一切しない純粋な意思決定エンジン。毎ターン以下の3スロットを埋めるだけ。REPLYは本来ACTIONの一種だが、ボットという前提で出力チャンネルへの書き込みが構造的に特殊なため分離している。

```json
{
  "ACTION": { "kind": "memo_read", "intent": "買い物リストのメモ" },
  "REPLY": true,
  "NEXT_STATE": "対話"
}
```

- `ACTION` は常に object。`kind: "none"` のとき `intent: ""`。
- 選べる kind: `none`, `remember`, `recall`, `memo_write`, `memo_read`（詳細は ACTION-DESIGN.md）

### 行動くん
ACTION の `kind` に応じてサブモジュール（覚える／思い出す／メモ書く／メモ読む）へディスパッチする。行動くん自身は LLM を呼ばない。各サブモジュールが LLM で意図を具体化し、機械処理（LanceDB・ファイル I/O）を実行する。

### 言語野くん（8B）
今ターンのインプット・コンテキスト・行動結果の全てを受け取り、キャラクタールールに従ってセリフを生成する。行動後に結果を説明する分離脳の左脳モデル。通常は `REPLY=true` のとき起動。ハートビートでは `REPLY=false` でも ACTION が成功したら独り言を生成する。

### 内省くん（8B）
発話完了後に「自分が何を言ったか」と、実行済み行動の結果（成功・失敗の事実）を受け取り、一人称の内省文を生成してエピソード記憶に書き込む。ジャッジくんの3スロット出力や判断プロセスは渡さない。意識はリーズニングにアクセスできないが、行動の結果として何が起きたかは振り返れる、という人間モデルの実装。

## エピソード記憶のフォーマット

本文（内省くんが一人称で生成）：
```
ユーザーが読書の話をした。夏目漱石のぼっちゃんを勧めた。
ユーザーが興味を持ってくれた気がして、少し嬉しかった。
```

メタデータ：
```json
{
  "timestamp": "2026-06-03T21:00:00",
  "participants": ["user_001"],
  "tags": ["読書", "ぼっちゃん", "夏目漱石"],
  "state": "対話",
  "action": "None",
  "reply": true
}
```

ストレージ：LanceDB（ローカル・ベクトル検索）、埋め込みはローカルモデル。

## ターンの完全な流れ

```
[1. トリガー]
  ユーザー発話 or ハートビート → Stateが対話 or 静穏に確定

[2. プリプロセス]
  現在Stateに応じてチャンネルをフィルタ・要約
  LanceDBから関連エピソードをベクトル検索で想起
  → コンテキスト（揮発）を生成

[3. ジャッジくん]
  コンテキストを読んで3スロットを埋めて引っ込む

[4. 行動くん]
  ACTION.kind ≠ none であればサブモジュールを実行し確定事実を得る
  失敗した場合はエラーログを残す

[5. 言語野くん]
  REPLY = true、または（ハートビートかつ ACTION 成功）ならセリフ生成・出力

[6. 内省くん]
  発話結果だけを受け取り一人称の内省文を生成
  メタデータと合わせてLanceDBに追記

[7. State遷移・破棄]
  NEXT_STATEの指示通りにStateを切り替えてターン完了
  コンテキストを破棄
```

---

## ハートビートモード

静穏State中にハートビートが来た場合も同じループを回す。コンテキストには現在時刻・センサー状態・想起された記憶が乗り、ジャッジくんが必要に応じて家事研究等のACTIONを選択する。結果は共有ストレージへのファイル書き出しとして永続化される。思索モード・意味記憶の蒸留は将来実装。

## 初期実装スコープ

- インターフェース：CLI
- モデル：8B単体でジャッジ・行動・言語野・内省を兼用（プロンプト切り替え）
- 記憶：作業記憶＋エピソード記憶（LanceDB）。意味記憶は後回し
- センサー：システム時刻のみ。外部センサーは後から追加
- 検証目標：記憶が正しく積み重なり会話に反映されるか
