# 自律ホームエージェント コンセプト設計書 v0.5

---

## コアコンセプト

「実行・言語化・内省」を構造的に分離し、小さいモデルでも破綻しにくくする最小認知アーキテクチャ。分離脳の左脳モデルに倣い、言語化と内省は行動の原因ではなく結果として生成される。意識はリーズニングプロセスにアクセスできず、行動と発話の結果だけを受け取って後付けで解釈する、という人間の認知モデルを設計原理とする。

単一の意思決定者（ジャッジ）は存在しない。各エージェントは同一の観測事実（TurnContext）から自律的に判断して動く。

### 認知の構造

ターン全体は **入力フェーズ** と **自律エージェントフェーズ** の 2 フェーズで構成する。

- **入力フェーズ（プリプロセス）**: センサー・永続記憶・作業記憶から揮発コンテキスト（`TurnContext`）を組み上げる。全エージェントが参照する事実の起点。フィルタは TurnContext に載せる量を絞るだけで、元データは変更しない。
- **自律エージェントフェーズ**: 各 actor が並列に自分で起動を判断し、起動した actor が自律実行する。actor 間の within-turn 依存はない。
  - **起動判定（multi-label）**: 判断系 actor（memo / webSearch / plan / synthesize）の起動を **1 コールでまとめて判定**する（各 actor の起動条件＝criteria を同時に見る）。単一の選定者ではなく、同一事実から各 actor の条件を並行評価する形。客観条件で一意に決まる起動（urlBrowse=URL の有無・recall=常時の背景想起）は機械ゲート（LLM を挟まない）
  - **actor pool**: 起動した各 actor が宣言した知覚チャンネルで TurnContext を参照し、自律実行して `ctx.actions` に積む
  - **language-agent**: 全チャンネル + `ctx.actions` を受け取り発話を生成する（State は言語野が宣言せず、観測事実から機械導出する）
  - **内省・内心更新**: 発話結果から内省を生成し LanceDB へ書き込む

### 知覚と作用 ― 受容器と効果器

入力（知覚）に**双対して、出力（作用＝世界への effect）も対称に置く**。**actor／faculty は世界への作用を、自分が持つ効果器（effector）で自分で起こす**。orchestrator が特定の作用を外から代行しない。

- **受容器（sensor）**＝感じる器官：時刻・会話・image_feed・audio_feed を知覚チャンネルへ（preprocess）。
- **効果器（effector）**＝作用する器官：**発話＝口**／記録(notes)＝手／カメラ運動＝首／外界探索＝MCP。各 action が run の中で自分の効果器を起こす。

発話も「口」という効果器の一つで、**特別な出力経路を持たない**（記録やカメラ運動と同じレイヤ）。＝**input=受容器 / output=効果器 の対称**。生物の受容器⟷効果器に倣う設計で、身体性（embodied）と「全部 actor」の土台になる。

帰結：発話は効果器が起こすので、**喋った後に内省・内心更新が走る**（発話＝ユーザー応答の単位／内省・affect は post-hoc・非ユーザー向け）。応答の体感は発話までで縮む（[EFFECTORS-PLAN.md](EFFECTORS-PLAN.md)・[ARCH-NEXT.md](ARCH-NEXT.md) §2.2）。

各 actor の起動の段はアクターごとに宣言する（none / systematic / llm・[ARCH-NEXT.md](ARCH-NEXT.md) §1.6）。段は「起動が**状況の判断**か**客観条件**か」で決まる。起動が状況の判断を要するアクター（webSearch「外の事実が要るか」・memory「想起か忘却か」等）は必ず LLM で判断し、キーワードやヒューリスティックでショートカット・スキップしない。起動が客観条件で一意に決まるアクター（recall=常時・視覚=画像の有無・distill=静穏idle）は機械ゲートでよい（LLM を挟む意味がない）。

## アーキテクチャ全体図

```
[外部センサー群]
  システム時刻 / その他IoT
       ↓ 常時書き込み
[永続層]
  エピソード記憶（LanceDB・ベクトル検索）
  意味記憶（エピソードからの蒸留・後回し）
  各センサー生ログ
       ↓ ターン開始時に想起・引用
[プリプロセス]
  State に応じて各チャンネルをフィルタ（元データは変更しない）
  全エージェントが読む TurnContext を組み上げる
       ↓
[TurnContext]（揮発・毎ターン生成・ターン終了後破棄）
  作業記憶（直近会話履歴）
  + 想起されたエピソード記憶
  + 必要な外部センサー状態
       ↓
[activate（各 actor 並列）]
  各 actor が mini-context（直近会話 + 内心 + actor リスト）× 小型モデルで
  自分の起動可否を 1 コールで判断（config / stateConfig で pool は事前フィルタ済み）
  → 各 actor が {"active": true/false, "intent": "..."} を返す
       ↓
[actor pool（並列実行）]
  起動した actor が各自の知覚チャンネルで TurnContext を参照
  memory（想起+忘却の受動記憶）/ memo（記録・読み書き統合）
  web-search / url-browse / webcam / plan / synthesize / ...
  結果を ctx.actions に積む
       ↓
[language-agent（言語野）]
  全 facts を受け取りキャラクタールールに従いセリフ生成
  発話するかどうかを自分で決める。NEXT_STATE も出力する
       ↓
[内省くん]
  発話結果と行動結果だけを受け取り一人称の内省文を生成
  各エージェントの判断プロセスもツールログも見えない
  LanceDB に追記（永続層へ）
       ↓
[内心ステート更新・State 遷移・TurnContext 破棄]
  NEXT_STATE の指示通りに State を切り替えてターン完了
```

## 主要コンポーネント詳細

### State
- `対話`：ユーザーが発話した、または自分が喋っている状態
- `静穏`：ハートビートによる潜伏状態
- `集中`：取り組み中の計画（ゴールノート）を進めるモード。Preprocess に計画チャンネルが常駐し、`plan` actor が進捗を記録する（詳細は DECISIONS.md §集中モード）

バリデーションはしない。逸脱値はログ出力して観察する。ログを参照して State 設計を拡張する。

### コンテキスト（揮発・使い捨て）
毎ターン開始時にプリプロセスが生成し、ターン終了後に破棄する一時的な作業台。人間のワーキングメモリに相当し、永続化しない。

流れ込むチャンネル：
- 作業記憶（直近の会話履歴）
- **内心**（持ち越す生の感情・余韻＝`affect` と、認知的焦点＝`concern` の2フィールド。内省が毎ターン書き換える。空 = 起きたて）。`concern` は actor の起動判断・想起クエリにも使う
- 想起されたエピソード記憶（LanceDBからベクトル検索で引用。直近ターンは除外し、再構成チャンネルとして使う）
- 意味記憶（夢で蒸留した知識）
- 外部センサー状態（初期はシステム時刻のみ）

**記憶とメモの違い**: エピソード記憶（LanceDB）はふんわり思い出す（LLM 要約・グラデーション可）。共有メモ（`data/notes/`）は意図して残した全文をそのまま保持し、読み出し時に LLM で改変しない。

**曖昧層と記録層の二層**: 感情・記憶（連想的・再構成的）は LLM の生成が担う＝小さいモデルでも良い反応をする。一方、エバ自身が保守する**構造化記録（plan 等）は精密さが要る**ので、構造・更新はコードが決定的に握り、LLM は「小さな操作（op）と中身の一文」だけ出す（強制ギプス）。精密さは曖昧層の外＝決定的な足場に置く——recall 側が「距離分類=機械＋LLM は意味だけ」なのと対称。人間も精密さは脳外の硬いツール（リスト・表）＋単純操作に逃がしている。詳細は DECISIONS.md §集中モード。

### プリプロセス
知的判断をしない濾過マシナリー。現在の State に応じて各チャンネルから必要な情報だけを引き出し、フィルタしてコンテキストを組み上げる。**フィルタは TurnContext に載せる量を絞るだけで、元データ（LanceDB・state.json）は変更しない**。フィルタ量は `stateConfig` で State 別に設定する。

### activate（各 actor の起動判断）
単一の選定者は置かない。各 actor が自分で起動可否を判断する軽量スクリーナーを持つ（`src/actors/activate.ts` の `createActivate` ファクトリ）。mini-context（直近 2〜3 ターン・最新発話・内心ステート・利用可能 actor リスト）のみを受け取り、1 コールで自分について `{active, intent}` を返す。想起済みエピソードは渡さない（それを取りに行くかを決めるのが activate のため）。config / stateConfig のフィルタで pool から除外済みの actor は判断自体が呼ばれない。false negative のコストが高いため迷ったら ON に倒す。actor（＝身体・能力）が増えても判断が分散しているので拡張しやすい。

### actor pool
activator が選んだ actor が並列に実行する。各 actor は宣言した**知覚チャンネル**の TurnContext フィールドのみを参照し、自律判断・実行して `ctx.actions` に `ActionOutcome` を積む。記憶系は**性質で2 faculty に分ける**（B'）: `memory`＝受動の記憶（recall+忘却を1 actorに統合・op で選ぶ・Read+Delete のみ）／`memo`＝能動の記録（notes の full CRUD・op＋連想する記憶の木）。他に `web-search` `url-browse` `webcam` `plan` `synthesize`（想起+外部+感性を統合して成果物を生成）。`remember` は廃止（意図的記憶は内心更新の importance 採点で扱う）。

### language-agent（言語野）
全 facts を受け取りキャラクタールールに従ってセリフを生成する。行動後に結果を説明する分離脳の左脳モデル。毎ターン必ず起動し、発話するかどうかを自分で決める。発話に加えて `NEXT_STATE` を出力し、State 遷移の責任を担う。

### 内省くん
発話完了後に「自分が何を言ったか」と、実行済み行動の結果（成功・失敗の事実）を受け取り、一人称の内省文を生成してエピソード記憶に書き込む。各エージェントの判断プロセスやツールログは渡さない。意識はリーズニングにアクセスできないが、行動の結果として何が起きたかは振り返れる、という人間モデルの実装。

内省の直後に**内心**（`affect`＝感情余韻 と `concern`＝認知的焦点）を1回の LLM 呼び出しで更新する（`updateAffectAndConcern`）。前の内心とこのターンの出来事を読み、心がどう動いたかを書き換える。忘却は数値減衰ではなく、内省と容量制約による創発（引きずるに足る強い理由がなければいまの落ち着きを書く）。`AgentState`（行動モード）とは別物。詳細は DECISIONS.md §affect/concern 分離設計。

## エピソード記憶のフォーマット

本文（内省くんが一人称で生成）：
```
ユーザーが読書の話をした。夏目漱石のぼっちゃんを勧めた。
ユーザーが興味を持ってくれた気がして、少し嬉しかった。
```

メタデータ：
```json
{
  "timestamp": "2026-06-03T21:00:00",
  "participants": ["user_001"],
  "tags": ["読書", "ぼっちゃん", "夏目漱石"],
  "state": "対話",
  "action": "None",
  "reply": true
}
```

ストレージ：LanceDB（ローカル・ベクトル検索）、埋め込みはローカルモデル。

## ターンの完全な流れ

```
[1. トリガー]
  ユーザー発話 or ハートビート → State が対話 or 静穏に確定

[2. プリプロセス]
  State に応じてチャンネルをフィルタ（元データは変更しない）
  LanceDB から関連エピソードをベクトル検索で想起
  → TurnContext（揮発）を生成

[3. activate（各 actor 並列）]
  config / stateConfig フィルタ済みの pool の各 actor が
  mini-context を読み、自分の起動可否を 1 コールで判断

[4. actor pool（並列）]
  起動した actor が各自の知覚チャンネルで TurnContext を参照
  自律判断・実行し ctx.actions に ActionOutcome を追加

[5. language-agent]
  全 facts を受け取りキャラクタールールに従い発話生成
  NEXT_STATE を決定して出力

[6. 内省くん]
  発話結果と行動結果だけを受け取り一人称の内省文を生成
  メタデータと合わせて LanceDB に追記

[7. 内心ステート更新・State 遷移・破棄]
  NEXT_STATE の指示通りに State を切り替えてターン完了
  TurnContext を破棄
```

## ハートビートモード

静穏 State 中にハートビートが来た場合も同じループを回す。コンテキストには現在時刻・センサー状態・想起された記憶が乗り、各エージェントが必要に応じて記憶操作や調査を自己判断する。idle heartbeat（全エージェントが何も積まず language-agent も発話しなかったターン）は内省・内心更新をスキップする。意味記憶の蒸留はバッチ `npm run dream` で実装済（エピソード→意味記憶）。思索モード・会話中の自発蒸留は将来実装。

## 初期実装スコープ

- インターフェース：CLI
- モデル：language / 内省 / 内心は `chatModel`（既定 `qwen3.6:35b-a3b`）、actor 実行は `actionModel`、actor の起動判断は `activatorModel`。`roles` / `actors[name].model` / `activatorModel` 設定でロール・actor 別に変更可
- 記憶：作業記憶＋エピソード記憶（LanceDB）。意味記憶は後回し
- センサー：システム時刻のみ。外部センサーは後から追加
- 検証目標：記憶が正しく積み重なり会話に反映されるか
