# 自律ホームエージェント コンセプト設計書 v0.5

---

## コアコンセプト

「実行・言語化・内省」を構造的に分離し、小さいモデルでも破綻しにくくする最小認知アーキテクチャ。分離脳の左脳モデルに倣い、言語化と内省は行動の原因ではなく結果として生成される。意識はリーズニングプロセスにアクセスできず、行動と発話の結果だけを受け取って後付けで解釈する、という人間の認知モデルを設計原理とする。

単一の意思決定者（ジャッジ）は存在しない。各エージェントは同一の観測事実（TurnContext）から自律的に判断して動く。

### 認知の構造

ターン全体は **入力フェーズ** と **自律エージェントフェーズ** の 2 フェーズで構成する。

- **入力フェーズ（プリプロセス）**: センサー・永続記憶・作業記憶から揮発コンテキスト（`TurnContext`）を組み上げる。全エージェントが参照する事実の起点。フィルタは TurnContext に載せる量を絞るだけで、元データは変更しない。
- **自律エージェントフェーズ**: memory・research・language の各エージェントが同一 TurnContext を読んで自己判断し、直列に実行する（本来は並行だが直列で近似）。前エージェントの結果は次エージェントから見える。
  - **memory-agent**: 記憶操作が必要か判断し、必要なら実行する
  - **research-agent**: 外部情報取得が必要か判断し、必要なら実行する
  - **language-agent**: 全 facts を受け取り発話を生成し NEXT_STATE を決める
  - **内省・内心更新**: 発話結果から内省を生成し LanceDB へ書き込む

各エージェントの活性化判断は必ず LLM で行う。ヒューリスティックによるスキップはしない。

## アーキテクチャ全体図

```
[外部センサー群]
  システム時刻 / その他IoT
       ↓ 常時書き込み
[永続層]
  エピソード記憶（LanceDB・ベクトル検索）
  意味記憶（エピソードからの蒸留・後回し）
  各センサー生ログ
       ↓ ターン開始時に想起・引用
[プリプロセス]
  State に応じて各チャンネルをフィルタ（元データは変更しない）
  全エージェントが読む TurnContext を組み上げる
       ↓
[TurnContext]（揮発・毎ターン生成・ターン終了後破棄）
  作業記憶（直近会話履歴）
  + 想起されたエピソード記憶
  + 必要な外部センサー状態
       ↓
[memory-agent]
  記憶操作（LanceDB / notes）が必要か 1 コールで判断・実行
  必要なければ即スキップ。facts を ctx.actions に積む
       ↓
[research-agent]
  外部情報取得（MCP）が必要か 1 コールで判断・実行
  memory facts も見える。必要なければ即スキップ
       ↓
[language-agent（言語野）]
  全 facts を受け取りキャラクタールールに従いセリフ生成
  発話するかどうかを自分で決める。NEXT_STATE も出力する
       ↓
[内省くん]
  発話結果と行動結果だけを受け取り一人称の内省文を生成
  各エージェントの判断プロセスもツールログも見えない
  LanceDB に追記（永続層へ）
       ↓
[内心ステート更新・State 遷移・TurnContext 破棄]
  NEXT_STATE の指示通りに State を切り替えてターン完了
```

## 主要コンポーネント詳細

### State
- `対話`：ユーザーが発話した、または自分が喋っている状態
- `静穏`：ハートビートによる潜伏状態

2値を指示するがバリデーションはしない。逸脱値はログ出力して観察する。ハートビート・思索モード実装時にログを参照してState設計を拡張する。

### コンテキスト（揮発・使い捨て）
毎ターン開始時にプリプロセスが生成し、ターン終了後に破棄する一時的な作業台。人間のワーキングメモリに相当し、永続化しない。

流れ込むチャンネル：
- 作業記憶（直近の会話履歴）
- **内心ステート**（持ち越す生の感情・余韻。内省が毎ターン書き換える。空 = 起きたて）
- 想起されたエピソード記憶（LanceDBからベクトル検索で引用。直近ターンは除外し、再構成チャンネルとして使う）
- 意味記憶（夢で蒸留した知識）
- 外部センサー状態（初期はシステム時刻のみ）

**記憶とメモの違い**: エピソード記憶（LanceDB）はふんわり思い出す（LLM 要約・グラデーション可）。共有メモ（`data/notes/`）は意図して残した全文をそのまま保持し、読み出し時に LLM で改変しない。

### プリプロセス
知的判断をしない濾過マシナリー。現在の State に応じて各チャンネルから必要な情報だけを引き出し、フィルタしてコンテキストを組み上げる。**フィルタは TurnContext に載せる量を絞るだけで、元データ（LanceDB・state.json）は変更しない**。フィルタ量は `stateConfig` で State 別に設定する。

### memory-agent
記憶操作（LanceDB・`data/notes/`）が必要か 1 コールで判断し、必要なら実行する。`activate: false` なら即終了。`activate: true` なら具体ツールと intent を決めて実行し `ctx.actions` に結果を積む。ツール: `remember` `recall` `forget` `memo_write` `memo_read` `distill`（スタブ）。

### research-agent
外部情報取得（MCP: web 検索・URL 閲覧・予定照会等）が必要か 1 コールで判断し、必要なら MCP ツールを実行する。memory-agent の facts も参照できる。`activate: false` なら即終了。

### language-agent（言語野）
全 facts を受け取りキャラクタールールに従ってセリフを生成する。行動後に結果を説明する分離脳の左脳モデル。毎ターン必ず起動し、発話するかどうかを自分で決める。発話に加えて `NEXT_STATE` を出力し、State 遷移の責任を担う。

### 内省くん
発話完了後に「自分が何を言ったか」と、実行済み行動の結果（成功・失敗の事実）を受け取り、一人称の内省文を生成してエピソード記憶に書き込む。各エージェントの判断プロセスやツールログは渡さない。意識はリーズニングにアクセスできないが、行動の結果として何が起きたかは振り返れる、という人間モデルの実装。

内省の直後に**内心ステート**を更新する。前の内心とこのターンの出来事を読み、心がどう動いたかを2〜3文で書き換える。忘却は数値減衰ではなく、内省と容量制約による創発（引きずるに足る強い理由がなければいまの落ち着きを書く）。`AgentState`（行動モード）とは別物。

## エピソード記憶のフォーマット

本文（内省くんが一人称で生成）：
```
ユーザーが読書の話をした。夏目漱石のぼっちゃんを勧めた。
ユーザーが興味を持ってくれた気がして、少し嬉しかった。
```

メタデータ：
```json
{
  "timestamp": "2026-06-03T21:00:00",
  "participants": ["user_001"],
  "tags": ["読書", "ぼっちゃん", "夏目漱石"],
  "state": "対話",
  "action": "None",
  "reply": true
}
```

ストレージ：LanceDB（ローカル・ベクトル検索）、埋め込みはローカルモデル。

## ターンの完全な流れ

```
[1. トリガー]
  ユーザー発話 or ハートビート → State が対話 or 静穏に確定

[2. プリプロセス]
  State に応じてチャンネルをフィルタ（元データは変更しない）
  LanceDB から関連エピソードをベクトル検索で想起
  → TurnContext（揮発）を生成

[3. memory-agent]
  TurnContext を読み、記憶操作が必要か 1 コールで判断
  必要なら実行し ctx.actions に ActionOutcome を追加

[4. research-agent]
  TurnContext（memory facts を含む）を読み、外部情報取得が必要か 1 コールで判断
  必要なら MCP ツールを実行し ctx.actions に ActionOutcome を追加

[5. language-agent]
  全 facts を受け取りキャラクタールールに従い発話生成
  NEXT_STATE を決定して出力

[6. 内省くん]
  発話結果と行動結果だけを受け取り一人称の内省文を生成
  メタデータと合わせて LanceDB に追記

[7. 内心ステート更新・State 遷移・破棄]
  NEXT_STATE の指示通りに State を切り替えてターン完了
  TurnContext を破棄
```

## ハートビートモード

静穏 State 中にハートビートが来た場合も同じループを回す。コンテキストには現在時刻・センサー状態・想起された記憶が乗り、各エージェントが必要に応じて記憶操作や調査を自己判断する。idle heartbeat（全エージェントが何も積まず language-agent も発話しなかったターン）は内省・内心更新をスキップする。思索モード・意味記憶の蒸留は将来実装。

## 初期実装スコープ

- インターフェース：CLI
- モデル：30B 相当を各エージェント（memory / research / language / 内省）で使用。`roles` 設定でロール別に変更可
- 記憶：作業記憶＋エピソード記憶（LanceDB）。意味記憶は後回し
- センサー：システム時刻のみ。外部センサーは後から追加
- 検証目標：記憶が正しく積み重なり会話に反映されるか
