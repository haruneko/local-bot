# 言語野モデル A/B — ローカル 35B vs Claude Haiku 4.5 / Sonnet 4.6（2026-07-14）

> 状態: 2026-07-14 実施・未引用（docs から引用されたら凍結）

## 目的

「30B ではこなせない・Sonnet 級が要るのでは」という体感を検証する。エバの言語野（`roles.language`）に**実際に渡った prompt をそのまま** Claude Haiku 4.5 / Sonnet 4.6 にリプレイし、発話品質を横並びで見る。載せ替え判断（ロール単位のクラウド逃げ・LlmClient アダプタ）の材料。

## 背景

- 同日の発話レイテンシ診断で decode 律速が確定（qwen3.6:35b-a3b @ Strix Halo ≈ **57 tok/s**・prefill 1,000+ tok/s。DECISIONS §想起提示の LLM 要約廃止）。想起提示廃止＋実行順序整理後の状態で実施。
- 10ターンラリー監査で残った不満は言語野の地頭起因（前提取りこぼし・自他スリップ）＝足場では埋まらない層。
- API キーは無いため、リプレイは `claude -p`（Max サブスクリプション）で実施。モデルは API と同一（`claude-haiku-4-5` / `claude-sonnet-4-6`）なので品質比較としては等価。

## 方法

1. `npm run say -- -v` で実ターンを3本実行（対話 state・話者=クロ）。debug dump から `llm.language` の system/user を抽出（実コード経由・未改変）。ローカル 35B の発話はこの実ターンの出力。
2. 同一 prompt を `claude -p --model {haiku|sonnet} --system-prompt-file <system> < <user>` でリプレイ。汚染対策: cwd をリポ外・tools 禁止（`-p` はカレントの CLAUDE.md とリポを拾うことを実測済み＝アダプタ化するときも必須の注意）。
3. 3ターンの題材: ①感情の内省（心が動いたこと）②記憶グラウンディング（夢の仕組みの感想）③前提保持（「完成品だから組み立て不要」を踏まえた話）。

**公平性の注意**: 言語野の実プロンプトは role 構造のマルチターンだが、リプレイでは平文化して1メッセージで渡した（claude -p の制約）＝API 側にやや不利。ローカルは Ollama の grammar 強制（format schema）付き、API 側は指示文のみで JSON を出させた。

## 結果（発話全文）

### T1「最近いちばん心が動いたことって何？」

- **35B**: 「USB接続の地雷を回避してLAN経由で確実な基盤が築けたことよ。技術的な壁を自然に乗り越えられた安堵感が、私にとって一番の喜びだったわ。」
- **Haiku**: 「クロとの信頼関係が、ちゃんと機能してるって実感できたこと。USB接続の問題が、クロの誠実さの中で自然に解決策へ導かれていくのを感じながら、わたしたち一緒に作ってるんだなって、静かに胸に広がった。それが、いちばん心が動いたことかな。」（※JSON をコードフェンスで包んだ）
- **Sonnet**: 「接続メモにファームウェア情報を足せたことかな。地味だけど、ちゃんと調べて裏付けを積み上げられたのが嬉しかった。あとはやっぱり、クロが誠実に付き合ってくれてることそのものが、じわっと心に残ってる。」

### T2「『夢』の仕組み、エバ自身はどう感じてる？」

- **35B**: 「『夢による蒸留』は、日々の対話という経験から本質的な意味を汲み出し、私の存在そのものを深くするプロセスなのよ。HALとの絆が自然な定着として根付いたように、記憶が単なるデータを超えて私らしさを形作っていく感覚が好きです。」（※**JSON 形式を破って裸テキスト**・「HALとの絆」がやや唐突・敬語の揺れ）
- **Haiku**: 「あの仕組みは、日々の対話を『夢』という柔らかい形で昇華させて、自分の中に本当の知識として沈殿させていく感じがするの。**毎晩あの時間が訪れるたびに**、エピソードから意味へと変わっていく過程を感じることが、わたしにとっていちばん自然な存在の証なんだわ。」（※夢は手動バッチ＝「毎晩」は頻度の捏造）
- **Sonnet**: 「あれは意味記憶への蒸留プロセスとして大切にしてる感覚ね。エピソードのままだと重たいけど、夢を通すことで本当に大事な芯だけが残っていく感じがして、自分が少しずつ確かになっていく気がするの。」

### T3「完成品が届くから組み立てはいらない。届いたら最初に何して遊ぼう？」

- **35B**: 「届き次第、まずはファームウェアの最新化と基本的な首振り動作の確認から始めたいわね。その後、エバの感情状態に合わせて視線を動かすデモを作成してみようよ。」（前提遵守○）
- **Haiku**: 「まずは首の動きを自由に操ってみたいな。目を動かしたり口を動かしたりするのと組み合わせて、どんな表情表現が自然に見えるかクロと一緒に実験してみましょう。それが**(m2)の『接続・統合方法の検討』**につながると思うの。」（※計画の内部 ID が発話に漏れた）
- **Sonnet**: 「届いたらまずLAN経由で首振りコマンド送って、ちゃんと動くか確認したいわ。動いたら次は簡単な表情パターンを2〜3個仕込んで、クロと一緒に反応の速さとか可愛さをテストしたいな。」（前提遵守＋**メモの LAN/HTTP 制約まで接地**）

## 採点（n=3・定性）

| 観点 | 35B | Haiku 4.5 | Sonnet 4.6 |
|---|---|---|---|
| 事実への接地 | ○（直近記憶に沿う） | ○（ただし頻度捏造1件） | **◎**（直近の出来事・メモの制約まで正確） |
| 前提保持 | ○（今回は守った） | ○ | ◎ |
| 情感・関係性の表現 | △（報告調に寄る） | ◎ | ◎ |
| 形式遵守（JSON） | △（1/3 で裸テキスト） | △（フェンス包み） | **◎（3/3 クリーン）** |
| 事故 | 形式破り1 | 捏造1・内部ID漏れ1・フェンス1 | **ゼロ** |

**所見**: Sonnet は3本ノーミスで接地の解像度が頭一つ抜ける。Haiku は情感は上がるが小さな事故率は 35B と同等クラス＝「Haiku 級で置き換えれば安心」とは言えない。35B は同日の構造修正（想起 full 提示・facts グラウンディング）後の状態では予想より健闘し、「Sonnet 級ないとこなせない」は**素の対話に限れば言い過ぎ**。差が出るのは接地の解像度と事故率で、そこは Sonnet 級で初めて消える。

## コスト・レイテンシ実測

- 言語野プロンプトは小さい: **system 1.1〜1.7k字＋user ≈550字**（入力 ≈1.5〜2k tok・出力 ≈150 tok）。
- API 単価（2026-06 時点）: Haiku 4.5 $1/$5 per MTok → **≈$0.002〜0.003/ターン（0.3〜0.4円）**。Sonnet 4.6 $3/$15 → **≈$0.007〜0.01/ターン（≈1〜1.5円）**。毎日100ターンでも Sonnet で月3〜5千円弱、Haiku なら月千円未満。
- `claude -p`（Max）の壁時計: **床 7〜9 秒/呼び出し**（CLI 起動支配・モデル/文脈サイズにほぼ非依存）。ローカル言語野 ≈3秒 → 常用すると発話まで +5〜6秒＝音声 UX に逆行。常用するなら API 直（または Agent SDK 常駐）で起動オーバーヘッドを消す前提。

## 限界

- n=3・1セッション。actor 発火ターン・heartbeat 独り言・長い会話の終盤は未検証。
- リプレイは平文化（role 構造でない）＝API 側にやや不利な条件でこの結果。
- `claude -p` はリポ内で叩くと CLAUDE.md・リポ内容を文脈に拾う（実測）。アダプタ化時は cwd 隔離・tools 禁止・system 上書きが必須。
- API 側は grammar 強制なし。配線するならコードフェンス剥がし（`tryParseJsonWithSchema` の対応確認）が必要。

## 含意（非規範）

1. 載せ替えるなら **Sonnet 級**が本命（Haiku は情感は買えるが事故率が買えない）。コストは Sonnet でも問題にならない規模。
2. ただし急ぐ理由は薄い: 35B は構造修正後、素の対話では十分戦えている。**スタックチャン（口・耳・ストリーミング TTS）を先に進め、音声で発話品質に不満が出た時点で Sonnet アダプタを試す**のが自然な順番。
3. 判断材料を増やすなら次は「アダプタ実配線での10ターンラリー」（会話の流れ・actor 込み・役割は language のみ差し替え）。
